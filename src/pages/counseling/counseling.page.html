UI/UX

I like to create clean and intuitive user interfaces. While not a creative myself, I have plenty of experience working with custom designs and UI templates. In my opinion, the secret of designing great user interfaces and giving the best user experience is knowing your audience and therefore, having a basic but firm understanding of the human psyche. This knowledge is dangerous though, as the more you understand, the more damage you can do without noticing. Therefore, my most important rule in UI/UX design is: always satisfy and never exploit.

I usually move on the Bootstrap - Tailwind axis when creating web interfaces. If there are no design plans available, I like to customize Bootstrap UI elements. If there are design plans or more complicated ideas, I usually stick with Tailwind. Also, I'm looking forward to trying Web Awesome for template UIs. In this area, I can help you with:

  • Implementing design (especially Figma) plans.
  • Creating clean and maintainable mobile-first responsive web UIs.
  • Improving user experience with slightly more complicated UI logic.
  • Improving user experience through gamification.

Infrastructure

Infrastructure is not my strongest suit. Not because I'm bad at it, I'm only not so interested in it. I got introduced to it through the stereotypic setup; there are no system administrators available, let the software developer take care of everything. I did, and I learned a lot about administering Linux systems on the way.

Lately, I'm more focused on the DevSecOps aspect of this area, especially on facets close to the architecture and development process, such as web security, CI/CD, automated testing, or containerization. While my experience in infrastructure is typically used for complementing other areas of expertise, if you need direct help from this area, I can help you with:

  • Installing and managing operating systems (preferably Linux) locally or remotely.
  • Traditional and spatial database documentation and design.
  • Creating CI/CD workflows and automated software tests.
  • Simple containerization with Docker.

Project management

I really like Agile. I like how early it recognized the importance of breaking up with traitional business-to-customer behavior in software development in favor of a client-centric one. Shortly after it came out, the Agile principles spread like wildfire. The IT industry showed how much it anticipated this change. What I especially like in this story is the lesson in it. It taught us, that money people, no matter how trained they are in economics, cannot control every aspect of every business. IT project management should be done with an IT background.

While project management is not something I would like to ever do full-time, I find value and satisfaction in teaching how to use version control properly with git. This not only includes proper collaboration techniques (like rebasing) but also using git-flow, semantic versioning, and creating a maintainable product lifecycle. In this area, I can help you with the following:

  • Implementing Agile in an unobtrusive way.
  • Using version control efficiently with git.
  • Creating a client-friendly product lifecycle and release cycle.
  • Choosing the right tools and methods, from experience.
